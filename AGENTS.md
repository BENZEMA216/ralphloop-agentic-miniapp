# 项目执行规则

本项目所有任务结束前都必须执行验证闭环。

## 结束前验证

每次任务结束前，必须根据相关需求文档、实施计划和验证文档检查任务是否已经完成。

默认规则：

1. 先识别本次任务的完成标准。
2. 运行能证明完成状态的验证命令。
3. 阅读完整输出，确认退出码和失败数量。
4. 如果验证失败、需求未覆盖或发现缺口，继续执行修复或补齐工作。
5. 只有在新鲜验证证据支持时，才能在最终回复里声明任务完成。

不得用“应该可以”“看起来可以”“理论上完成”代替验证。

## 最终回复

最终回复必须说明：

- 实际完成了什么。
- 运行了哪些验证命令。
- 验证结果是什么。
- 如果仍有未完成项，必须明确列出，并继续执行可在当前回合完成的部分。

## 适用范围

该规则适用于：

- 文档任务。
- 代码实现。
- 调研和计划。
- 本地环境准备。
- 测试、构建、运行和发布前检查。
