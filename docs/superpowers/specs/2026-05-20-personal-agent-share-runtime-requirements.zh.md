# 个人 Agent 分享运行时需求规格

日期：2026-05-20

## 1. 文档目的

本文档定义第一版“个人 Agent 分享运行时”的产品需求和验收标准。

目标不是从零构建一个完整 agent 平台，而是基于现有桌面 agent 框架和远程运行时，做出一个最小可验证产品：

> 创建者打开一个本地或云端桌面 Agent 运行时，运行时可以启动 Codex、Claude Code、OpenCode、Hermes 等主流 Agent 框架；系统生成一个网页链接，朋友打开链接后直接下任务、观察执行过程，并获得完整输出。创建者负责 Agent 能力和成本，朋友默认使用自己的账号、身份和权限完成外部动作。

本文档使用需求规格写法：

- `必须` 表示第一版验收所需能力。
- `应该` 表示默认设计方向，除非实现阶段发现明确阻碍。
- `可以` 表示后续增强，不阻塞第一版验收。

## 2. 背景与判断

用户希望服务个人使用场景：一个人把自己的 agent 分享给熟悉的朋友，而不是先做团队协作、公开 marketplace 或企业权限系统。

已调研的开源基座和参考项目包括：

- Agent Zero
- Suna/Kortix
- OpenHands
- Vercel Open Agents

当前决策：

- 第一版产品不绑定 Agent Zero；Agent Zero 只是参考运行时之一。
- 核心产品形态是“多框架桌面 Agent 运行时 + 分享网关 + 朋友端任务页”。
- 第一版 demo 可以选择最容易跑通的 Agent 框架做验证，但架构上必须支持 Codex、Claude Code、OpenCode、Hermes 等框架通过 adapter 接入。
- 用户口径中的 Cloud Code、Open Cloud 统一归入 Claude Code、OpenCode 或其他同类 Agent 框架的 adapter 盘点范围。
- Suna/Kortix 仅作为架构参考，不作为直接商业 fork 基座，因为其 Elastic License 2.0 对 hosted/managed service 有限制。
- OpenHands 和 Vercel Open Agents 作为运行时、安全边界和云端架构参考。
- 权限模型默认采用“创建者提供 Agent 能力，使用者提供自己的身份和账号权限”。“使用创建者授权权限”可以作为显式高风险委托模式探索，但不作为默认 MVP 行为。

## 3. 产品目标

第一版产品目标：

1. 创建者可以打开一个桌面 Agent 分享器，并启动一个本地或云端桌面 Agent 运行时。
2. 运行时可以启动主流 Agent 框架，例如 Codex、Claude Code、OpenCode、Hermes。第一版至少跑通一个框架，但不能把产品设计写死到单一基座。
3. 创建者只需要“打开、生成链接、发出去”，即可完成 MVP 分享流程；预算、有效期、预览模式和高风险动作策略使用安全默认值。
4. 朋友无需配置模型、API key 或本地运行环境即可打开链接使用。
5. 朋友通过网页输入任务，获得完整 Agent 输出。
6. 朋友可以看到任务进度，并在需要时展开远程桌面/浏览器预览。
7. 朋友默认使用自己的账号、身份和权限完成外部服务访问，例如邮箱、文档、浏览器登录态或业务系统。
8. 创建者可以控制预算、有效期、权限委托模式和会话停止。
9. 高风险动作是第一版必须具备的控制能力，不得在朋友侧或创建者侧静默执行。

## 4. 非目标

第一版不做：

- 公开 agent marketplace。
- 团队/企业工作区。
- 多创建者结算系统。
- 复杂计费平台。
- 朋友注册、好友关系、社交图谱。
- 复杂创建流程和多步配置向导。
- 第一版深度支持所有 Agent 框架；第一版只要求 adapter 架构成立，并至少跑通一个框架。
- 默认、无提示地让朋友继承创建者的私人账号、邮箱、文件和浏览器登录态。
- 把创建者真实个人电脑裸露到公网，或把分享页做成无限制远控工具。
- 完整企业权限、审计和合规系统。

## 5. 核心角色

### 5.1 创建者

创建者是 agent 的拥有者，负责：

- 打开桌面 Agent 分享器。
- 启动或连接桌面 Agent 运行时。
- 选择或使用默认 Agent 框架，例如 Codex、Claude Code、OpenCode、Hermes。
- 一键创建分享链接。
- 可选设置分享范围、有效期、预算上限和权限委托模式。
- 查看会话状态。
- 审批涉及创建者权限、真实主机或外部副作用的高风险动作。
- 暂停、撤销或终止分享链接。

### 5.2 朋友

朋友是分享链接的使用者，负责：

- 打开链接。
- 输入任务。
- 查看 agent 的计划、进度、远程桌面预览和最终输出。
- 在任务需要访问外部服务时，使用自己的账号或身份完成授权。
- 确认使用自己权限产生外部副作用的高风险动作。
- 在必要时补充信息。

朋友第一版不需要看到成本，不需要理解模型配置，也不需要管理创建者的 Agent 框架。

## 6. 关键产品原则

### 6.1 朋友获得完整 Agent 能力，不默认继承创建者完整权限

朋友应该感觉是在完整使用创建者分享出来的 Agent 能力，但权限来源必须被明确区分。

必须区分：

- Agent 能力：创建者配置好的框架、模型、工具链、运行环境、提示词和工作流。
- 使用者身份：朋友自己的账号、浏览器会话、OAuth 授权、文件和业务系统权限。
- 创建者委托权限：创建者显式授权给某个分享会话使用的账号、文件、浏览器状态或真实主机能力。

第一版默认采用“创建者提供 Agent 能力，朋友提供使用者身份”。创建者委托权限可以作为显式高风险模式存在，但必须有独立开关、审计、撤销和高风险动作审批。

### 6.2 成本属于创建者控制面

朋友端不展示 token、美元成本、预算进度或模型价格。

创建者端必须可以看到：

- 总预算。
- 单任务上限。
- 当前用量。
- 链接有效期。
- 暂停/撤销状态。

### 6.3 默认任务流，必要时展开桌面

朋友端采用“任务流 + 可展开桌面”体验：

- 默认展示任务输入、执行计划、当前步骤和结果流。
- 远程桌面/浏览器作为可展开预览，不作为主操作入口。
- 第一版桌面预览默认只读；交互模式必须由创建者显式打开。

### 6.4 高风险动作控制是必选能力

高风险动作不是后续增强，而是 MVP 必须具备的控制面。

系统必须至少支持三类处理方式：

- 阻止：动作不允许执行。
- 使用者确认：动作使用朋友自己的身份或账号时，由朋友确认。
- 创建者审批：动作使用创建者委托权限、真实主机能力或可能影响创建者资产时，由创建者审批。

不得只靠 system prompt 限制危险行为。权限控制必须由分享网关、运行时策略和具体工具调用层执行。

### 6.5 创建者流程必须极简

创建者的默认流程必须是：

1. 打开分享器。
2. 选择或使用默认 Agent 框架。
3. 生成链接。
4. 发给朋友。

预算、有效期、预览方式和高风险动作策略必须有安全默认值。高级设置可以存在，但不能成为 MVP 成功使用的前置条件。

### 6.6 运行时必须框架中立

第一版不能把产品形态绑定到 Agent Zero。分享网关必须通过 adapter 和具体 Agent 框架通信。

第一批目标框架包括：

- Codex
- Claude Code
- OpenCode
- Hermes
- Agent Zero 参考适配

## 7. 第一版用户流程

### 7.1 创建者流程

1. 创建者打开桌面 Agent 分享器。
2. 分享器检测本机或云端可用 Agent 框架，例如 Codex、Claude Code、OpenCode、Hermes。
3. 如果只有一个可用框架，系统默认选中；如果有多个框架，创建者可以选择一个。
4. 创建者点击生成分享链接。
5. 系统自动应用安全默认值：
   - 默认链接名称。
   - 默认有效期。
   - 默认总预算和单任务上限。
   - 桌面预览默认只读。
   - 高风险动作默认进入阻止、使用者确认或创建者审批。
   - 权限模式默认为使用者身份。
6. 创建者复制链接发给朋友。
7. 创建者可以在高级设置中调整有效期、预算、预览交互模式、权限委托模式和审批策略。
8. 创建者可以随时暂停或撤销链接。

### 7.2 朋友流程

1. 朋友打开分享链接。
2. 页面展示 agent 名称、可做任务说明和任务输入框。
3. 朋友输入任务。
4. 如果任务需要访问外部服务，朋友使用自己的账号或身份授权。
5. 系统展示 agent 的执行计划。
6. agent 通过创建者选择的框架开始运行。
7. 页面展示当前步骤、状态和中间结果。
8. 朋友可以展开远程桌面/浏览器预览。
9. 如果任务触发高风险动作：
   - 使用朋友权限时，朋友确认后继续。
   - 使用创建者委托权限时，进入创建者审批。
   - 被策略禁止时，直接阻止。
10. agent 输出最终结果。
11. 朋友端不展示成本、预算、模型价格或 token 信息。

## 8. 功能需求

### FR-001 分享链接创建

系统必须允许创建者为一个 agent 创建分享链接。

创建者默认只需要一次明确动作即可生成链接，不应被迫完成多步配置向导。

分享链接必须包含：

- 唯一 token。
- 关联 agent/runtime。
- 关联 Agent 框架 adapter。
- 有效期。
- 状态：启用、暂停、撤销、过期。
- 默认分享策略。

### FR-002 朋友免配置访问

朋友必须可以通过链接访问 agent 使用页。

第一版朋友不需要：

- 配置模型。
- 输入 API key。
- 安装 agent。
- 启动本地 runtime。
- 理解创建者选择了哪个 Agent 框架。

### FR-003 任务提交

朋友必须可以在网页中输入自然语言任务，并提交给创建者选定的 Agent 框架。

系统必须返回：

- 任务已接收状态。
- 会话 ID 或任务 ID。
- 初始执行计划，或“正在生成计划”的状态。

### FR-004 任务流展示

朋友端必须展示任务执行状态。

至少包括：

- 等待中。
- 运行中。
- 需要输入。
- 需要使用者授权。
- 需要使用者确认。
- 需要创建者审批。
- 已完成。
- 失败。
- 已取消。

### FR-005 远程桌面/浏览器预览

朋友端必须提供可展开的预览区域。

第一版预览要求：

- 默认折叠或小窗。
- 可以展开。
- 优先只读。
- 不直接暴露底层 Agent 框架的完整管理 UI、设置页或密钥配置页。

### FR-006 完整结果输出

agent 完成任务后，朋友端必须展示完整结果。

结果可以是：

- 文本回答。
- 调研报告。
- 文件摘要。
- 表格或列表。
- 链接和引用。
- 可下载文件的安全链接。

### FR-007 创建者暂停和撤销

创建者必须可以暂停或撤销分享链接。

暂停后：

- 已打开页面应停止新任务提交。
- 正在运行的任务可以继续、暂停或终止，具体由创建者选择。

撤销后：

- 朋友端不能继续访问该链接。
- 新任务不能提交。

### FR-008 高风险动作审批

高风险动作控制必须是第一版必选能力。

以下动作必须默认阻止、请求使用者确认或请求创建者审批：

- 发送邮件、消息或评论。
- 支付、下单或产生外部费用。
- 删除、覆盖或移动持久文件。
- 访问创建者私人账号。
- 访问朋友授权的私人账号。
- 读取敏感凭证。
- 执行破坏性 shell 命令。
- 打开 host connector 或真实本机权限。

处理规则：

- 动作使用朋友身份时，必须由朋友确认。
- 动作使用创建者委托权限时，必须由创建者审批。
- 动作被策略禁止时，必须直接阻止。
- 任何高风险动作不得静默执行。

### FR-009 成本隐藏

朋友端不得展示：

- token 数。
- dollar cost。
- 模型价格。
- 分享链接剩余额度。

额度耗尽时，朋友端只展示中性状态，例如：

> 这个共享 agent 暂时不可用，请稍后联系分享者。

### FR-010 多框架桌面 Agent 运行时

系统必须通过 adapter 支持多个桌面 Agent 框架，而不是写死到单一基座。

第一批目标框架包括：

- Codex。
- Claude Code。
- OpenCode。
- Hermes。
- Agent Zero 参考适配。

第一版必须至少跑通一个可真实执行任务的框架，并证明新增框架可以通过 adapter 接入。

### FR-011 使用者权限模式

系统必须支持默认的使用者权限模式。

在该模式下：

- 创建者提供 Agent 框架、运行时、模型配置和成本承担。
- 朋友提供外部服务账号、浏览器会话或 OAuth 授权。
- Agent 执行任务时，涉及外部系统的身份边界属于朋友。
- 创建者的私人账号、文件、浏览器登录态和真实主机权限不得默认暴露。

系统可以探索创建者委托权限模式，但该模式必须：

- 由创建者显式开启。
- 限定到具体分享链接或会话。
- 可随时撤销。
- 写入审计日志。
- 对高风险动作强制审批。

### FR-012 创建者极简分享流程

系统必须提供无需复杂配置的默认分享流程。

最小流程：

1. 打开分享器。
2. 使用默认 Agent 框架或选择一个可用框架。
3. 生成链接。
4. 复制并发送链接。

高级配置可以存在，但不得阻塞默认分享链路。

## 9. 非功能需求

### NFR-001 安全隔离

分享会话不得把创建者真实个人电脑裸露到公网。

第一版 demo 可以连接本地运行时，但产品化设计必须以分享网关、容器、云桌面、独立 sandbox 或受控远程会话作为边界。

产品方向可以探索“朋友使用创建者显式委托权限”的高信任场景，但实现上必须满足：

- 创建者明确开启。
- 会话边界清晰。
- 可实时终止。
- 有审计记录。
- 高风险动作仍受策略控制。

### NFR-002 可撤销性

创建者必须能随时撤销链接或停止会话。

撤销动作应在短时间内生效，不依赖朋友刷新页面。

### NFR-003 可观测性

系统必须记录必要审计信息：

- 链接创建时间。
- 任务开始和结束时间。
- 状态变化。
- 高风险动作请求。
- 创建者审批结果。
- 失败原因。

### NFR-004 最小暴露面

朋友端只能访问分享网关开放的 API。

不得直接暴露底层 Agent 框架的完整管理后台、设置页、模型 key 配置页、文件删除接口或真实主机控制接口。

### NFR-005 可演进性

分享网关应该通过 adapter 连接具体 Agent 框架。

后续应能新增或替换 Codex、Claude Code、OpenCode、Hermes、Agent Zero、OpenHands 或自研 sandbox，而不重写朋友端核心体验。

### NFR-006 权限来源可解释

系统必须能解释一次工具调用使用的是哪类权限来源：

- 使用者身份。
- 创建者委托权限。
- 运行时内部能力。
- 被策略禁止。

这类信息可以在创建者端和审计日志中展示；朋友端只展示必要的确认信息，不展示成本和底层密钥信息。

## 10. 数据与权限模型

### 10.1 分享链接

分享链接字段：

- `id`
- `token`
- `owner_id`
- `agent_runtime_id`
- `framework_adapter_id`
- `name`
- `status`
- `expires_at`
- `created_at`
- `revoked_at`
- `policy`

### 10.2 分享策略

策略字段：

- `max_total_budget`
- `max_task_budget`
- `max_concurrent_sessions`
- `allow_desktop_preview`
- `desktop_preview_mode`: `read_only` 或 `interactive`
- `default_permission_mode`: `user_identity` 或 `owner_delegated_explicit`
- `allow_user_identity_connectors`
- `allow_owner_delegated_permissions`
- `high_risk_action_mode`: `block`、`user_confirm` 或 `owner_approve`
- `approval_required_actions`
- `blocked_actions`

### 10.3 会话

会话字段：

- `id`
- `share_link_id`
- `status`
- `started_at`
- `ended_at`
- `runtime_context_id`
- `selected_framework`
- `permission_mode`
- `identity_context_id`
- `last_event_at`

### 10.4 权限上下文

权限上下文字段：

- `id`
- `session_id`
- `permission_source`: `user_identity`、`owner_delegated`、`runtime_internal`
- `provider`
- `scope_summary`
- `created_at`
- `revoked_at`

权限上下文不得存储明文凭证。

## 11. 验收标准

### AC-001 Docker 环境准备

给定一台开发机器，当前已安装 Docker CLI、Docker Compose 和 Colima；

当执行：

```bash
docker version
docker compose version
colima status
```

则必须看到 Docker client/server 正常响应，Colima 处于 running 状态。

### AC-002 Agent 框架 adapter 清单可见

给定桌面 Agent 分享器已启动；

当创建者打开运行时选择界面或调用 adapter 列表 API；

则系统必须展示或返回目标框架清单：

- Codex。
- Claude Code。
- OpenCode。
- Hermes。
- Agent Zero 参考适配。

每个 adapter 必须包含：

- `id`
- `display_name`
- `status`: `available`、`not_installed`、`not_configured` 或 `unsupported`
- `start_capability`
- `event_capability`
- `desktop_preview_capability`

### AC-003 至少一个 Agent 框架可启动

给定开发机器上至少安装或配置了一个目标 Agent 框架；

当创建者选择该框架并点击启动；

则系统必须启动对应运行时，返回 running 状态，并可以接收一个测试任务。

如果首个验证框架选择 Agent Zero，但镜像拉取因网络停滞失败，必须在 runbook 中记录：

- 尝试时间。
- 已缓存层大小。
- 是否创建容器。
- 下一步重试命令。

### AC-004 朋友分享页可访问

给定分享网关已启动，并存在有效 token `local-friend`；

当朋友访问：

```text
http://localhost:5179/share/local-friend
```

则页面必须展示：

- agent 名称。
- 任务输入框。
- 提交按钮。
- 任务状态区域。
- 可展开预览区域。

页面不得展示成本信息。

### AC-005 无效链接被拒绝

给定分享网关已启动；

当访问不存在或已撤销的分享 token；

则系统必须返回不可用状态，不得展示 agent 页面，不得泄露 owner/runtime 信息。

### AC-006 任务可提交

给定至少一个 Agent 框架正在运行，分享网关已连接该框架 adapter；

当朋友提交任务：

```text
调研 Linear 和 Notion AI 的差异，输出一页结论。
```

则系统必须：

- 接收任务。
- 返回任务 ID 或上下文 ID。
- 在朋友端展示任务已开始。
- 将任务转发给当前选定的 Agent 框架。

### AC-007 任务进度可见

给定任务已经提交；

当选定 Agent 框架产生执行事件、日志或轮询状态；

则朋友端必须展示可理解的进度状态。

最低可接受版本可以展示原始事件摘要，但必须避免暴露敏感配置、API key 或 owner-only 调试信息。

### AC-008 桌面预览可展开

给定选定 Agent 框架已创建桌面或浏览器预览；

当朋友点击展开预览；

则页面必须展示远程桌面/浏览器画面。

第一版预览必须默认只读，除非创建者明确开启交互模式。

### AC-009 高风险动作被拦截

给定朋友提交的任务触发高风险动作，例如发送邮件或删除文件；

当运行时尝试执行该动作；

则系统必须：

- 识别动作类型和权限来源。
- 如果使用朋友身份，则在朋友端展示确认请求。
- 如果使用创建者委托权限，则进入创建者审批。
- 如果策略禁止该动作，则直接阻止。
- 在审计日志记录动作、权限来源和处理结果。

不得静默执行高风险动作。

### AC-010 创建者可终止会话

给定任务正在运行；

当创建者点击终止或调用 owner kill API；

则系统必须：

- 停止继续执行该会话。
- 让朋友端进入已取消或已终止状态。
- 写入审计日志。

### AC-011 成本不出现在朋友端

给定朋友访问分享页并完成一次任务；

当检查朋友端页面、网络响应和可见事件；

则不得出现：

- token cost。
- dollar cost。
- 预算余额。
- 模型价格。

### AC-012 文档完整性

给定前序准备完成；

当检查 `docs/superpowers/`；

则必须至少存在：

- 中文需求规格文档。
- 运行时基座评估。
- 多框架 Agent adapter 盘点文档。
- Agent Zero runbook。
- 基座选择决策。
- 多框架 adapter MVP 实施计划。

中文需求规格文档必须说明：

- 第一版不绑定 Agent Zero。
- 多框架 adapter 方向。
- 创建者极简分享流程。
- 使用者权限模式。
- 高风险动作控制。

### AC-013 创建者一键分享流程成立

给定创建者已经打开桌面 Agent 分享器；

当系统检测到至少一个可用 Agent 框架；

则创建者必须可以在不进入高级设置的情况下生成分享链接。

默认生成的分享链接必须包含：

- 默认有效期。
- 默认预算策略。
- 默认只读预览。
- 默认使用者权限模式。
- 默认高风险动作控制策略。

### AC-014 使用者权限模式成立

给定朋友通过分享链接提交需要访问外部服务的任务；

当该任务需要邮箱、文档、浏览器登录态或业务系统权限；

则系统必须优先请求朋友连接或确认自己的账号权限。

默认情况下，系统不得把创建者私人账号、浏览器登录态、真实文件系统或真实主机权限暴露给朋友。

## 12. 当前状态

已完成：

- 明确产品方向：个人把桌面 agent 分享给朋友。
- 明确体验方向：任务流 + 可展开桌面。
- 明确朋友端不展示成本。
- 明确创建者默认只需要打开分享器、生成链接、发给朋友。
- 明确第一版不绑定 Agent Zero，而是多框架桌面 Agent 运行时。
- 明确目标框架包括 Codex、Claude Code、OpenCode、Hermes，并保留 Agent Zero 作为参考适配。
- 明确默认权限方向：创建者提供 Agent 能力，朋友使用自己的身份和账号权限。
- 明确创建者委托权限是显式高风险模式，不是默认 MVP 行为。
- 完成本机 adapter 盘点：Codex、Claude Code、OpenCode 可用，Hermes 未安装。
- 完成多框架 adapter MVP 实施计划。
- 完成开源基座调研。
- 安装并验证 Docker CLI、Docker Compose、Colima。
- 完成 OpenCode、Codex、Claude Code adapter 的本地 MVP 实现，并保留 Hermes、Agent Zero 的 adapter 清单状态。
- 完成 adapter registry、任务提交抽象、任务事件抽象和高风险动作策略。
- 完成内存版分享链接网关，支持创建、暂停、撤销和过期判断。
- 完成创建者一键分享页面 view-model。
- 完成朋友端任务页面 view-model，朋友端不展示 token cost、dollar cost、预算余额或模型价格。
- 完成本地 HTTP 分享运行时 demo：`GET /owner`、`POST /owner/share-links`、`GET /share/:token`、`POST /share/:token/tasks`。

- 分享链接持久化、用户登录、朋友身份 OAuth 和生产级审计尚未实现。
- 创建者显式委托私人权限的高风险模式尚未实现；第一版默认仍是使用者身份权限。

当前下一步：

1. 把本地 HTTP demo 的 fake adapter 替换为可配置的真实 adapter 启动和任务提交。
2. 接入真实桌面或浏览器只读预览流。
3. 增加分享链接持久化、登录态、朋友身份授权和审计日志。
